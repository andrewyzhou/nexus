Generate relationship edges between S&P 500 companies using 4 different methods.

Prerequisites: Tasks 4-5 complete (DB schema + REST API with dummy data working).
Depends on: Task 6a (Backend team) for scraper/data/processed/nodes.json
Produces: scraper/data/processed/edges.json (consumed by backend team for DB seeding)

  scraper/data/processed/nodes.json — produced by the backend scraper team (Task 6a)

================================================================================
EDGE TYPE 1: INDUSTRY PEERS (2 people)
================================================================================

File: ai/pipeline/industry_peers.py

Logic:
  - Group companies by their "industry" field
  - Within each industry group, create edges between all pairs
  - Weight by market cap proximity:
      weight = 1.0 - abs(log(cap_a) - log(cap_b)) / max_log_diff
      (normalize so weights fall in 0.0-1.0 range)
  - Companies in the same industry but vastly different sizes get lower weight
  - Companies in the same sector but different industry: optional weaker edges
    (weight = 0.3 * market_cap_proximity)

Output per edge:
  {
    "source_id": "AAPL",
    "target_id": "MSFT",
    "relationship_type": "industry_peer",
    "weight": 0.85,
    "metadata": {
      "industry": "Consumer Electronics",
      "sector": "Technology",
      "reason": "Same industry: Consumer Electronics"
    }
  }

Handle:
  - Companies with null industry: skip (do not create edges)
  - Industries with only 1 company: no edges to create
  - Avoid duplicate pairs: if (A, B) exists, do not also create (B, A)

Expected volume: ~5,000-15,000 edges depending on industry group sizes.


================================================================================
EDGE TYPE 2: FINANCIAL SIMILARITY (2 people)
================================================================================

File: ai/pipeline/financial_similarity.py

Logic:
  - Extract the financial_metrics vector for each company:
      [pe_ratio, pb_ratio, ps_ratio, beta, profit_margin,
       institutional_holdings, short_ratio, week52_change]
  - Handle nulls: impute with column median, or skip companies with >50% null metrics
  - Normalize each metric to z-scores (subtract mean, divide by std dev)
  - Compute pairwise cosine similarity between all company vectors
  - Only keep edges where cosine_similarity > 0.7 (threshold — tune as needed)
  - Weight = cosine_similarity value (already 0.0-1.0)

Output per edge:
  {
    "source_id": "AAPL",
    "target_id": "GOOGL",
    "relationship_type": "financial_similarity",
    "weight": 0.82,
    "metadata": {
      "cosine_similarity": 0.82,
      "metrics_used": ["pe_ratio", "pb_ratio", "ps_ratio", "beta",
                        "profit_margin", "institutional_holdings",
                        "short_ratio", "week52_change"],
      "reason": "High financial metric similarity (cosine=0.82)"
    }
  }

Libraries: numpy, scikit-learn (sklearn.metrics.pairwise.cosine_similarity)

Performance note:
  - ~500 companies = 124,750 pairs. Computing all is fine in memory.
  - Use numpy vectorized operations, not nested loops.
  - Filter to only edges above threshold before writing to JSON.

Expected volume: ~10,000-30,000 edges depending on threshold.


================================================================================
EDGE TYPE 3: NLP SIMILARITY (2 people)
================================================================================

File: ai/pipeline/nlp_similarity.py

Logic:
  - Use company "description" field from nodes.json
  - Apply TF-IDF vectorization on all descriptions
  - Compute pairwise cosine similarity on TF-IDF vectors
  - Only keep edges where similarity > 0.3 (descriptions are short, so lower threshold)
  - Weight = cosine_similarity value

Steps:
  1. Collect all descriptions into a list (skip companies with null/empty description)
  2. TF-IDF vectorize:
       from sklearn.feature_extraction.text import TfidfVectorizer
       vectorizer = TfidfVectorizer(stop_words='english', max_features=5000)
       tfidf_matrix = vectorizer.fit_transform(descriptions)
  3. Compute pairwise cosine similarity:
       from sklearn.metrics.pairwise import cosine_similarity
       sim_matrix = cosine_similarity(tfidf_matrix)
  4. Extract edges above threshold

Output per edge:
  {
    "source_id": "AAPL",
    "target_id": "SAMSUNG",
    "relationship_type": "nlp_similarity",
    "weight": 0.45,
    "metadata": {
      "cosine_similarity": 0.45,
      "method": "tfidf",
      "reason": "Similar business descriptions (cosine=0.45)"
    }
  }

Libraries: scikit-learn

Handle:
  - Companies with null/empty description: exclude from NLP analysis
  - Very short descriptions (<20 chars): exclude
  - Ensure source_id < target_id alphabetically to avoid duplicate pairs

Expected volume: ~5,000-20,000 edges depending on threshold.


================================================================================
EDGE TYPE 4: LLM-POWERED RELATIONSHIPS (2 people)
================================================================================

File: ai/pipeline/llm_relationships.py

Logic:
  - Use GPT-4 or Claude API to identify real-world business relationships
  - Focus on: competitor, supplier, customer, parent, subsidiary
  - Process companies in batches to manage API costs

Approach:
  1. Pre-filter: only process companies with market_cap in top 100 for initial run
     (expand later if budget allows)
  2. For each company, prompt the LLM with company name + sector + industry
  3. Ask: "What are the key business relationships for {company}?"
  4. Parse structured response into edges

Prompt template (example):
  """
  Given the following S&P 500 company:
    Name: {name}
    Ticker: {ticker}
    Sector: {sector}
    Industry: {industry}
    Description: {description}

  And the following list of other S&P 500 companies:
    {list of tickers and names}

  Identify business relationships between {name} and other companies in the list.
  For each relationship, provide:
    - related_ticker: the ticker of the related company
    - subtype: one of "competitor", "supplier", "customer", "parent", "subsidiary"
    - confidence: 0.0 to 1.0
    - reason: brief explanation

  Return as JSON array.
  """

Output per edge:
  {
    "source_id": "AAPL",
    "target_id": "QCOM",
    "relationship_type": "llm_relationship",
    "weight": 0.90,
    "metadata": {
      "subtype": "supplier",
      "reason": "Qualcomm supplies modem chips for iPhones",
      "model": "gpt-4",
      "confidence": 0.90
    }
  }

Implementation:
  - Use OpenAI API (GPT-4) or Anthropic API (Claude) — team's choice
  - Request structured JSON output (use function calling or JSON mode)
  - Rate limiting: respect API rate limits, add delays between calls
  - Caching: save raw LLM responses to ai/data/llm_cache/ to avoid re-querying
  - Cost management: estimate cost before running full batch
      (~100 companies * 1 prompt each ≈ ~$5-15 with GPT-4)
  - Validate: only create edges where both tickers exist in nodes.json
  - Weight = LLM's confidence score

Handle:
  - LLM hallucinations: validate all tickers against the nodes.json ticker list
  - Duplicates: if LLM says A→B and B→A for same relationship, keep only one
  - API failures: retry with backoff, skip company after 3 failures

Expected volume: ~500-2,000 edges (smaller set but high quality).


================================================================================
PIPELINE ORCHESTRATION
================================================================================

File: ai/pipeline/generate_edges.py (main entry point)

This script runs all 4 edge generators and merges the results:

  1. Load nodes.json
  2. Run industry_peers.py       → list of edges
  3. Run financial_similarity.py → list of edges
  4. Run nlp_similarity.py       → list of edges
  5. Run llm_relationships.py    → list of edges
  6. Merge all edges into single list
  7. Deduplicate: if same (source_id, target_id, relationship_type) appears
     multiple times, keep the one with highest weight
  8. Validate all source_id and target_id exist in nodes.json
  9. Write to scraper/data/processed/edges.json
  10. Print summary:
        "Generated edges:"
        "  Industry peers:        8,432"
        "  Financial similarity: 14,219"
        "  NLP similarity:        6,891"
        "  LLM relationships:     1,247"
        "  Total:                30,789"

Usage:
  python -m ai.pipeline.generate_edges
  python -m ai.pipeline.generate_edges --skip-llm   (skip expensive LLM step)
  python -m ai.pipeline.generate_edges --only industry_peers  (run single type)

Each sub-pipeline should also be runnable standalone:
  python -m ai.pipeline.industry_peers
  python -m ai.pipeline.financial_similarity
  python -m ai.pipeline.nlp_similarity
  python -m ai.pipeline.llm_relationships

COORDINATION WITH BACKEND (Task 6a)

The data flows like this:

  [Backend scraper team]           [AI team]                  [Backend DB team]
  scraper/scraper.py          -->  ai/pipeline/          -->  backend/db/seed.py
       |                               |                           |
  scraper/data/raw/*.json          Reads nodes.json,          Reads both files,
       |                           generates edges             seeds PostgreSQL
  scraper/data/processed/               |
       nodes.json             -->  scraper/data/processed/
                                        edges.json

Timing:
  1. Backend scraper team produces nodes.json FIRST — wait for this before running
  2. AI team can develop/test with mock nodes.json (see below) while waiting
  3. Once nodes.json is ready, run full pipeline to produce edges.json
  4. Backend DB team seeds both files into PostgreSQL

Mock data for development:
  Create a small test file ai/tests/mock_nodes.json with 10-20 companies for
  development and testing. Use real-ish data so edge generators behave realistically.

nodes.json contract (what you receive):
  - Array of objects, each with: id, ticker, name, sector, industry, market_cap,
    description, and financial_metrics object
  - financial_metrics contains: pe_ratio, pb_ratio, ps_ratio, beta,
    profit_margin, institutional_holdings, short_ratio, week52_change
  - Null values are possible for any field — handle gracefully

edges.json contract (what you produce):
  - Array of objects, each with: source_id, target_id, relationship_type, weight, metadata
  - source_id and target_id are tickers (must match company id in nodes.json)
  - relationship_type is one of: "industry_peer", "financial_similarity",
    "nlp_similarity", "llm_relationship"
  - weight is float 0.0-1.0
  - metadata is an object (contents vary by edge type, see sections above)
  - No duplicate (source_id, target_id, relationship_type) triples
